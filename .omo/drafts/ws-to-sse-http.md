# Draft: WebSocket → SSE + HTTP 迁移 (/t 页面麻将)

## Requirements (confirmed)
- 当前 /t 页面麻将流程使用 WebSocket，体验很慢
- 之前流程复杂所以用 WS，现在流程简化为"选场选风"
- 迁移到 SSE + HTTP 架构
- 改为**非服务端阻塞流程**

## 阻塞 vs 非阻塞决策
- **阻塞（SSE）**：选场 - 所有用户进入同一规则后阻塞等待
- **阻塞（HTTP）**：上传分数
- **非阻塞**：选风、录入分数数字确认

## 选风流程变更
- **当前**：服务端校验临时用户是否绑定手机、绑定公式战（阻塞流程）
- **改为**：
  - 选择规则后，不符合条件的用户直接展示"不符合"
  - 提供对应按钮，点击弹出：
    - 绑定手机号弹窗
    - 绑定公式战弹窗
    - 两个一起绑定的弹窗
    - 登录弹窗
  - 不再阻塞服务端
